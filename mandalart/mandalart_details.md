# 나의 만다라트: 64가지 상세 실천 계획

**중심 목표: 의료 바이오 도메인 데이터 사이언티스트 되기**

---

## 1. 데이터 과학 역량 및 기술 숙련도

*   [ ] **AI 심화 학습 및 적용:** 현재 수강 중인 AI 강의 내용을 단순 수강을 넘어, 핵심 알고리즘(예: Transformer) 원리 깊이 이해 및 코드 재현 시도. 배운 내용을 의료/바이오 관련 미니 프로젝트에 적용해보기.
*   [ ] **Python 라이브러리 숙련도 향상:** Pandas, NumPy, Scikit-learn 고급 기능(예: MultiIndex, 벡터화 연산, 파이프라인 구축) 학습 및 실습 (공식 문서, 관련 서적 활용, 주 2시간).
*   [ ] **SQL 쿼리 능력 강화:** 기본적인 조회를 넘어, 윈도우 함수, CTE, 서브쿼리 최적화 등 중급/고급 SQL 문제 풀이 연습 (온라인 플랫폼 활용 - LeetCode SQL, HackerRank 등, 주 1시간).
*   [ ] **알고리즘/자료구조 기초 다지기:** 프로그래머스/백준 등에서 레벨 1~2 수준의 알고리즘/자료구조 문제 매일 1문제 이상 풀기 습관화. 필요시 관련 온라인 강의 병행 학습.
*   [ ] **머신러닝 모델링 파이프라인 정립:** 데이터 전처리-특성 공학-모델 선택/학습-평가-튜닝으로 이어지는 표준적인 ML 모델링 파이프라인을 자신만의 코드로 정립하고, 여러 데이터셋(Kaggle 등)에 반복 적용하며 익숙해지기.
*   [ ] **탐색적 데이터 분석(EDA) 강화:** 테이블 데이터 분석 시, 시각화 및 통계적 기법을 활용하여 데이터의 패턴, 이상치, 변수 간 관계를 깊이 있게 탐색하고 그 과정을 체계적으로 문서화하는 연습 (모든 개인 프로젝트 필수 포함).
*   [ ] **Git/GitHub 활용 생활화:** 모든 코드 작성 및 프로젝트 진행 시 Git으로 버전 관리 철저히 하고, GitHub 리포지토리에 꾸준히 push하는 습관. (Branch, Merge, Commit 메시지 컨벤션 등 학습 포함)
*   [ ] **코드 리뷰 및 클린 코드 연습:** 자신의 코드를 스스로 리뷰하며 개선점(가독성, 효율성, 재사용성)을 찾는 연습을 하고, 클린 코드 관련 자료(책, 블로그)를 학습하여 좋은 코드 작성 원칙 적용 노력 (월 1회).

## 2. 의료/바이오 데이터 분석 전문성

*   [ ] **핵심 생명과학 개념 구조화:** 칸 아카데미 등 활용하여 학습한 기초 생물학/화학 지식을 바탕으로, 주요 개념(예: 세포 신호 전달, 유전자 편집 기술 원리, 면역 반응 기전) 간의 관계를 시각화(마인드맵 등)하거나 설명 노트를 작성하여 지식 체계 구축 (월 1회 이상).
*   [ ] **임상/연구 용어 맥락 기반 해석:** 논문, 임상 가이드라인, 의학 뉴스 등에서 접하는 주요 의료/바이오 용어를 단순 정의 암기를 넘어, 해당 문맥에서의 정확한 의미와 함의(implication)를 파악하고 해석하는 연습 (주 2회 이상 접하는 용어 대상).
*   [ ] **신약 개발 데이터 생태계 매핑:** 신약 개발 단계별(탐색~허가)로 생성/활용되는 데이터 종류, 주요 분석 목적, 관련 규제 요건(예: FDA 제출 데이터) 등을 조사하여 하나의 '데이터 생태계 맵'으로 시각화 및 정리 (분기별 1회 업데이트).
*   [ ] **생존 분석 모델 가정 검토 및 결과 비판적 해석:** 생존 분석 모델(Kaplan-Meier, Cox 등) 적용 시, 모델의 통계적 가정이 데이터에 부합하는지 검토하는 방법을 학습하고, 결과(HR, p-value 등)의 임상적/통계적 유의성과 한계를 비판적으로 해석하는 능력 함양 (관련 스터디/프로젝트 시 필수 적용).
*   [ ] **표준 의료 데이터(OMOP-CDM 등) 구조 탐구:** 국제 표준 공통 데이터 모델(예: OMOP-CDM)의 설계 철학, 주요 테이블 구조, 용어 표준화 방식(SNOMED CT, RxNorm 등)을 학습하고, 실제 데이터 변환 사례 탐구 (분기별 1회).
*   [ ] **유전체 분석 파이프라인 논리 이해:** 기본적인 유전체 분석(예: RNA-Seq DE 분석, GWAS) 파이프라인의 각 단계별(QC-Alignment-Quantification-Statistical Test 등) 목적과 기본 원리를 이해하고, 각 단계에서 사용되는 주요 도구/알고리즘의 선택 기준 학습.
*   [ ] **임상 연구 논문 비판적 읽기 (방법론 중심):** 관심 분야의 주요 임상 연구 논문(RCT 등) 선정하여, 연구 설계, 대상자 선정/제외 기준, 통계 분석 방법, 결과 해석의 타당성 및 잠재적 비뚤림(Bias)을 평가하는 '비판적 읽기(Critical Appraisal)' 연습 (월 1회).
*   [ ] **특정 질병/치료 영역 '미니 전문가' 되기:** 관심 질병/치료 영역(예: 특정 유형 면역항암제)에 대해, 관련 생물학적 기전부터 임상 데이터 분석 사례, 최신 연구 동향까지 깊이 있게 학습하여, 해당 주제에 대해 체계적으로 설명할 수 있는 수준(동료에게 발표 가능 수준)을 목표로 학습 및 자료 정리 (반기별 1개 주제 선정).

## 3. 데이터 기반 가치 창출 능력

*   [ ] **프로젝트 목표 설정 시 '기대 가치' 명시:** 개인 프로젝트 시작 단계에서 기술적 목표와 더불어, '이 프로젝트가 성공적으로 수행된다면 어떤 문제를 해결하거나 어떤 긍정적 변화(기대 가치)를 가져올 수 있는가?'를 구체적으로 정의하고 문서화.
*   [ ] **분석 결과 기반 '실행 제안' 도출 연습:** 분석 결과 요약 시 (5번 요소 연계), 인사이트(So What?)에 더해 '이 결과를 바탕으로 당장 혹은 미래에 무엇을 시도해볼 수 있는가?(Now What?)' 에 대한 구체적인 실행 아이디어나 제안을 1가지 이상 포함시키는 연습.
*   [ ] **분석 결과의 '잠재적 임팩트' 기술:** 개인 프로젝트 결과 보고(README 등) 시, 분석 결과의 정량적/정성적 잠재적 영향력(예: 특정 질병 예측 정확도 개선으로 인한 기대 효과, 신약 후보 탐색 시간 단축 가능성 등)을 추정하고 그 근거와 함께 명확하게 기술하는 연습.
*   [ ] **'가치 스토리라인' 중심 포트폴리오 구성:** GitHub 등에 프로젝트를 정리할 때, 사용한 기술 나열을 넘어 '문제(Problem) → 접근(Approach) → 결과(Result) → 창출된 가치/인사이트(Value/Insight) → 제안(Proposal)' 의 스토리가 명확히 드러나도록 구성하고 작성 (4번 요소 연계).
*   [ ] **현업 데이터 활용 '가치 창출' 사례 분석:** 의료/바이오 분야 기업/연구소의 데이터/AI 활용 성공 사례(뉴스, 기술 블로그, 발표 자료 등)를 찾아보고, '어떤 데이터로 어떤 문제를 해결하여 구체적으로 어떤 가치(비용 절감, 효율 증대, 신규 서비스 등)를 창출했는지' 분석하여 기록 (월 1회).
*   [ ] **모델 성능 지표의 '실질 가치' 연결:** 모델 성능 평가 시 기술적 지표(Accuracy, AUC 등)와 함께, '이 지표가 1% 개선되는 것이 실제 현장(예: 병원, 연구실)에서는 어떤 의미(가치)를 갖는가?' 를 고민하고 설명하는 연습 (관련 프로젝트 진행 시).
*   [ ] **'데이터 기반 솔루션' 아이데이션:** 현재 학습 중인 도메인 지식/기술을 바탕으로, 의료/바이오 분야의 미충족 수요(Unmet needs)나 비효율을 해결할 수 있는 새로운 '데이터 기반 솔루션 아이디어' 를 구체적으로 구상하고, 핵심 가치 제안(Value Proposition) 을 명확히 정의해보는 연습 (분기별 1회).
*   [ ] **프로젝트 '핵심 가치' 엘리베이터 피치 연습:** 개인 프로젝트 결과나 솔루션 아이디어를 바탕으로, 1분 안에 '이것이 왜 중요하고 어떤 핵심 가치를 제공하는지' 명료하게 설명하는 엘리베이터 피치(Elevator Pitch) 스크립트 작성 및 말하기 연습 (월 1회).

## 4. 설득력 있는 소통 및 다학제 협업

*   [ ] **"쉬운 글쓰기" 연습 (블로그/GitHub 활용):** 학습한 기술/개념(예: 특정 ML 모델, 통계 용어)에 대해, 의료/바이오 배경지식이 없는 사람을 대상으로 쉽고 간결하게 설명하는 짧은 글(500자 내외)을 월 1회 이상 작성하여 개인 블로그나 GitHub에 게시. (기술 용어는 최소화하거나 쉽게 풀어서 설명)
*   [ ] **의료 전문가 관점의 글 읽기:** 의료 전문가 대상 매체(국내외 의학 저널 뉴스, 의료 전문 언론 기사, 병원/학회 소식 등)에서 데이터/AI에 대해 다룬 글을 주 1회 이상 찾아 읽고, 그들이 사용하는 용어, 관심사, 우려 사항 등을 파악하여 기록하기.
*   [ ] **데이터 시각화 '설명력' 강화:** 개인/스터디 프로젝트의 시각화 단계에서, (Python 라이브러리, BI 툴 등 활용하여) 차트 설계(유형 선택, 디자인)부터 해석 및 설명 문구 작성까지 전 과정에 걸쳐 '비전공자 대상의 명확한 메시지 전달'을 최우선 목표로 삼고 결과물을 만드는 연습.
*   [ ] **GitHub README '스토리텔링' 적용:** 개인 프로젝트 README 작성 시, 기술적 나열뿐 아니라 '어떤 문제를 풀려고 했는지(Why)', '어떤 접근 방식을 사용했고 그 이유는 무엇인지', '핵심 결과는 무엇이며 어떤 의미가 있는지(So What?)' 를 명확히 드러내는 스토리텔링 구조로 작성 연습.
*   [ ] **상황별 분석 결과 요약 연습:** 특정 실무 상황(보고 대상(청중), 목적, 긴급성 등 포함)을 가정하여, 분석 결과를 그 상황에 맞춰 핵심만 간결하게 요약(1페이지 분량)하고 중요한 정보(시사점, 한계점, 다음 단계 제언 등)를 포함하여 전달하는 연습 (분기별 1회).
*   [ ] **적극적 경청 및 질문 노트:** 논문/기술문서/강의 학습 시, 단순히 내용을 받아 적는 것을 넘어 '이해가 안 되는 부분', '더 알고 싶은 질문' 을 명확하게 기록하는 습관 들이기 (향후 커뮤니티 질문 또는 스터디 토론 시 활용).
*   [ ] **발표/강연 분석 및 학습:** 데이터 과학/AI 관련 유명 컨퍼런스(NeurIPS, ICML 등) 또는 TED 등에서 복잡한 내용을 명료하게 전달하는 발표 영상을 월 1회 이상 시청. 발표자가 어떤 구조, 비유, 시각 자료를 사용해서 청중의 이해를 돕는지 분석하고 핵심 전달 전략을 요약 기록.
*   [ ] **발표 논리 구조 설계 및 개요 작성 연습:** 학습/프로젝트 결과를 다학제 청중에게 발표한다고 가정하고, (1) 핵심 메시지 정의, (2) 논리적 흐름(스토리라인) 구상, (3) 슬라이드별 핵심 내용(개요) 작성 단계를 거쳐 핵심 메시지 중심의 5-10장 분량 발표 구조 설계 및 개요 문서(텍스트 기반)를 만드는 연습 (분기별 1회).

## 5. 분석적 사고 및 문제 해결 능력

*   [ ] **데이터 문제 해결 프레임워크 학습 및 적용:** '문제 정의 → 가설 설정 → 데이터 수집/분석 → 결론 도출 → 실행 제안' 등 데이터 기반 문제 해결 프레임워크 (예: CRISP-DM 변형)를 학습하고, 개인 프로젝트 진행 시 각 단계를 의식적으로 적용 및 기록.
*   [ ] **프로젝트 '핵심 질문' 정의 연습:** 개인 프로젝트 시작 시, '이 분석을 통해 궁극적으로 답하고 싶은 핵심 질문(Key Question)은 무엇인가?' 를 명확히 정의하고, 모든 분석 과정이 이 질문에 답하는 데 집중되도록 노력.
*   [ ] **분석 전 '가설 목록' 작성 습관:** 본격적인 데이터 탐색/분석 전에, 검증하고자 하는 가설(Hypothesis)을 최소 3개 이상 구체적으로 설정하고, 각 가설 검증에 필요한 데이터와 분석 방법을 미리 계획하는 연습.
*   [ ] **결과 해석 시 'So What?' 분석:** 분석 결과(차트, 모델 성능 등) 도출 후, 반드시 '그래서 이 결과가 의미하는 바(Insight)는 무엇인가?', '이 결과로부터 어떤 다음 단계(Action)를 제안할 수 있는가?' 를 스스로에게 묻고 구체적인 답변을 정리하는 습관.
*   [ ] **우수 분석 사례 '논리 구조' 학습:** 우수 Kaggle 솔루션이나 잘 쓰인 분석 논문/보고서를 읽을 때, 단순히 기법이나 결과만 보는 것이 아니라 '어떤 논리적 흐름(Logic Flow)으로 문제를 정의하고 해결했는지', '결론을 뒷받침하는 핵심 근거는 무엇인지' 를 파악하고 분석 (월 1회).
*   [ ] **문제 구조화 도구 활용 연습 (Logic Tree 등):** 복잡한 문제(예: 특정 질병 예측 모델 성능 개선 방안)에 대해, 문제를 상호 배타적이고 전체 포괄적인(MECE) 하위 요소로 분해하는 Logic Tree 또는 Issue Tree를 직접 그려 문제 구조를 명확히 파악하는 연습 (분기별 1회).
*   [ ] **'왜?' 질문으로 근본 원인 탐색:** 분석 중 예상과 다른 결과나 문제가 발생했을 때, 표면적인 이유에 만족하지 않고 '왜(Why)?'라는 질문을 최소 3번 이상 반복하여 근본적인 원인(Root Cause)을 찾아가는 연습 (필요시 적용).
*   [ ] **자신의 분석 '자가 검증' 습관:** 주요 분석 결론이나 모델 설계 완료 후, 의도적으로 자신의 분석 과정과 결과에 대해 반론(가정 오류, 데이터 한계, 다른 해석 가능성 등)을 제기하며 타당성을 스스로 검증하고 보완하는 연습 (주요 프로젝트 완료 시).

## 7. 성장 마인드셋 및 자기 관리

*   [ ] **일일/주간 학습 회고 및 '내일 더 잘하기' 계획:** 하루 마무리 또는 매주 일요일 저녁, 그날/그 주의 학습/활동을 간단히 돌아보며, '잘한 점', '배운 점', 그리고 '내일(다음 주)은 어떻게 더 잘할 수 있을지' 구체적인 액션 플랜 1가지 이상 기록하기 (GitHub 또는 개인 노트).
*   [ ] **'성장 기회로서의 실패' 분석 (취사선택 연습):** 코드가 막히거나 예상치 못한 결과가 나왔을 때, 감정이 가라앉은 후 '여기서 배울 점 딱 하나는 무엇인가?', '다음에 다르게 시도해볼 것 딱 하나는 무엇인가?' 를 찾아 기록. 나머지는 '이번엔 이게 최선이었어' 라고 인정하고 털어버리기 연습.
*   [ ] **주간 우선순위 설정 및 현실적 시간 계획:** 매주 일요일 저녁, 다음 주 만다라트 목표 중 가장 중요한 1~2가지 선정하고, 현재 스케줄 고려하여 실현 가능한 학습/실천 시간을 캘린더에 미리 확보하기. (무리하지 않는 자기 관리)
*   [ ] **최소 건강 루틴 설정 및 준수 노력:** '이것만은 지키자'는 최소한의 건강 목표 설정 (예: 하루 6시간 이상 자기, 주 3회 20분 걷기, 매일 스트레칭 5분). 완벽하지 않아도 꾸준히 지키려고 노력하기.
*   [ ] **선배/동료 인사이트 탐색:** 관심 분야(의료/바이오 데이터 사이언스) 선배/동료들의 성장 과정, 어려움 극복 경험 등을 담은 블로그 회고글, 인터뷰 기사, 발표 영상 등을 월 1회 이상 찾아보며 간접적으로 배우고 동기 부여 얻기.
*   [ ] **'나의 왜' 와 만다라트 정기 점검 (월 1회):** 매월 첫째 주 주말, 내가 왜 의료/바이오 데이터 사이언티스트가 되고 싶은지 정리했던 글을 다시 읽고, 지난달 만다라트 진행 상황을 돌아보며 방향성 재확인 및 동기 부여 재충전.
*   [ ] **작은 성공/성장 기록 및 자기 격려:** 매주 만다라트 계획 중 하나라도 달성했거나, 스스로 생각하기에 '이 부분은 어제보다 나아졌다'고 느낀 점 1가지 찾아 구체적으로 기록하고, 스스로에게 긍정적인 피드백 해주기.
*   [ ] **약점 집중 보완 및 자기 설명 연습:** 매월 초, 가장 부족한 기술/지식 영역 1가지 선정. 해당 월에는 관련 문제 해결 시 바로 검색/질문 전에 문제 상황, 시도, 예상/결과 등을 글로 정리하거나 소리 내어 설명하며 스스로 해결 시도 (주 1회 이상 적용). 이 과정에서 약점 보완을 위한 구체적인 연습/학습 병행.
